## 3. 작업 A — 위험점수의 학술·제도적 재정립 [피드백 ①]

### A-1. 문제
현재 `scoring.ts`는 5개 신호를 0~1로 정규화 후 **집단별 가중치를 곱해 합산(0~100)** 한다. 가중치 숫자(예: elderly power=25)와 임계값(65/45)의 **출처가 없다.** 심사단은 정확히 이 점을 때렸다. 두 갈래로 근거를 보강한다: **(A-2) 점수의 *형식*** 은 임상 노쇠지수로, **(A-3) 신호의 *선택*** 은 한국 복지 발굴 위기정보 체계·실서비스로.

### A-2. 점수 형식의 근거 — Frailty Index(결핍누적 모델)

복지 위험점수의 형식적 뼈대로 **노쇠지수(Frailty Index, FI)의 결핍누적(deficit accumulation) 방식**을 채택한다.

- **원장원. 노쇠의 최신지견. 대한의사협회지 2022;65(2):108-114.** doi:10.5124/jkma.2022.65.2.108 — Rockwood의 Frailty Index 소개. 핵심: **여러 결핍 항목을 각각 0/1(또는 부분점수)로 만들고, 전체 항목 중 결핍 비율로 점수를 산출한다**(예: 70개 항목 중 7개 → 0.1). 점수↑ = 노쇠↑. 한국형 일차의료 노쇠지수(KFI-PC)는 56개 항목이며 위험요인에 **사회활동 부족·인지저하 등 비신체 영역까지 포함**. URL: https://jkma.org/upload/pdf/jkma-2022-65-2-108.pdf

**이 프레임이 주는 정당성:**
1. "여러 신호를 0~1 결핍으로 정규화 → 합쳐 0~100 위험지수"가 **임상 검증된 표준(FI)과 동형(isomorphic)**. → "0~100 위험지수 형식 자체가 정당한가"에 대한 답.
2. **고전 FI는 가중치 없는 단순 평균(결핍 개수/전체)**. 즉 *가중치를 안 써도 되는* 근거 기반 베이스라인이 존재한다. 이게 "가중치를 막 정함" 비판을 정면 해소한다.

### A-3. 신호 선택의 근거 — 한국 복지 발굴 위기정보 체계 + 실서비스

> 우리가 쓰는 5개 신호가 *왜 위험신호인가*는 임상논문만으로는 약하다. 그러나 **한국 정부가 실제로 위기가구 발굴에 쓰는 신호 체계**와 **이미 운영 중인 공공 안부살핌 서비스**가 강한 제도적 근거를 준다.

- **보건복지부 복지 사각지대 발굴 시스템(사회보장급여법 기반):** 18개 기관에서 **위기정보 39종 → 44종**(단전·단수·건강보험료 체납·공공요금 체납·금융연체·임차료/관리비 3개월↑ 체납·고용위기·의료비 등)을 입수, **빅데이터 분석으로 위기 가능성 상위 약 4%(회차당 약 16~20만명, 전체 약 500만명 중)** 를 선별해 지자체에 통보 → 방문·전화 확인. (출처: 정책브리핑·복지부 보도자료, §7-7.)
  - → **우리 신호(전력 급감·진료 단절·통신 연체·우편·고립)는 이 공식 위기정보 체계의 *대리/근사(proxy)*** 다. 신호 선택의 정당성을 여기에 맞춰 서술한다.
  - → **수원·송파 세 모녀 사건**: 이 시스템이 *놓친* 사례(예: 건보료 체납 정보로 방문했으나 실거주지 불일치·연락두절). 프로젝트의 도입 훅과 직접 연결 — "기존 발굴 체계의 사각/실패를 보조 레이어로 메운다"는 본 프로젝트의 정체성을 뒷받침.
- **한국전력 + 이동통신 3사(SKT·KT·LGU+) 'AI 안부살핌' 서비스 / 서울시 스마트플러그 사업:** 전력·통신 데이터로 1인가구 이상 징후를 감지해 지자체 담당자에게 알림. (출처 §7-8.) **두 가지가 직접 입증됨:** ① 전력·통신이 *실제로* 쓰이는 비침습 위기신호다(우리 power·telecom 신호 정당), ② 한전이 **"전력 급감만 보던 기존 방식 → 생활패턴 전반 분석·개인별 패턴 업데이트로 변화신호 포착"** 으로 발전시켰다 → **우리가 §4에서 하려는 패턴기반 이상탐지가 실서비스의 실제 진화 경로와 동일**.

### A-4. 공식 재설계 — 2단 구조 (베이스라인 + 집단특화 확장)

> 아래 구조로 `scoring.ts`를 재작성한다. 상수는 전부 §3 근거 또는 §4 ML로 정당화되도록 주석을 단다.

**(1) 베이스라인 = 인용 가능한 순수 Frailty Index (가중치 없음)**
```
각 신호 → 결핍 점수 deficit_i ∈ [0,1]   // 0=결핍없음, 1=완전결핍
baseFI   = mean(deficit_i)               // Rockwood 결핍누적: 단순 평균
riskBase = round(100 × baseFI)
```
- 임의 가중치 0개 → "근거 없음" 비판에서 자유. UI/문서에 "기본 점수는 Rockwood Frailty Index(결핍누적) 방식"이라고 출처와 함께 명시.

**(2) 집단특화 확장 = 문헌으로 *방향·상한이 제약된* 가중치 (프로젝트 차별점 ②)**
```
riskWeighted = round(100 × Σ(deficit_i × w_i^group) / Σ w_i^group)
```
- `w_i^group`는 "막 정한 값"이 아니라 다음 제약을 받는다:
  - **문헌/제도 방향 제약**: 각 신호의 가중치 크기를 §A-5 근거표의 "근거 강도"가 제한(강=높은 가중 허용, 약=낮은 가중 강제).
  - **데이터 정합 제약(권장)**: §4 ML이 산출하는 신호 기여도와 모순되지 않게 조정. 학습된 값도 G1·G5에 따라 "공식 상수"로 고정·주석화.
- **베이스라인↔확장을 UI 토글**로 비교(기존 "집단 모형 전환" 토글에 "무가중 FI" 옵션 추가) → 심사단에게 가중치 유무 차이를 투명하게 시연.

**(3) 임계값(밴드)**
- 현재 65/45는 임의값 → **분위수 기반(예: 합성 모집단 상위 X%)** 또는 명시적 가정으로 교체하고 근거 주석. (참고: 실제 복지부 시스템도 "상위 약 4%"식 분위수 선별을 쓴다 → 분위수 컷의 제도적 선례.)

### A-5. 신호별 근거표 (정규화·가중 처리 지침)

> Claude Code는 아래 출처를 **직접 확인**하고 각 신호 주석에 근거를 단다. "근거 강도"는 가중치 상한 결정에 쓴다. **약한 신호에 강한 인용을 달지 말 것.**

| 신호 | 정규화(현행) | 근거 출처 | 근거 강도 | 가중·처리 지침 |
|---|---|---|---|---|
| **전력 급감** power | `min(급감률/100,1)` | ① 한전·통신3사 안부살핌 + 서울시 스마트플러그(실서비스, §7-8) ② arXiv:2407.00524(전력 기반 비침습 모니터링·이상탐지) | **강(실서비스+논문)** | 핵심 신호. 단 실서비스가 "단순 급감→패턴변화"로 이동했듯, **"급감"만 점수화하지 말고 §4 이상탐지의 입력**으로도 쓴다. 정량 위험식은 논문에 없으므로 변환식은 가정 주석. |
| **진료 단절** medical | `min(단절일/365,1)` | 박채린·김현우(2025) "1인 가구 의료수요 발생·미충족 결정요인", 보건사회연구 45(1):221-239 (§7-3) | **중** | "진료 단절 = 의료접점 장기단절 + 미충족 의료"로 결핍화. 365일 등 임계는 임의값 → 근거/가정 주석 필수. |
| **통신 연체** telecom | `min(연체월/3,1)` | ① 위기정보 체계의 각종 요금 체납(경제취약 위기신호) 계열 ② 한전·통신3사 서비스가 **통신 데이터**를 위기신호로 사용(§7-7,8) | **중** | "경제취약 + 연락채널 단절"의 복합신호로 결핍화. ⚠️ *통신비 체납(arrears)* 이 명시적 44종 중 하나라고 단정하진 말 것(위기정보의 통신 요소는 주로 *연락처 연계*임). 대신 "요금 체납류 경제신호 + 실서비스의 통신 데이터 활용"으로 서술. (개선안: 연체 대신 **통신 사용패턴 변화**를 신호로 쓰면 실서비스에 더 근접 → §8.) |
| **우편 미수령** mail | `min(미수령주/4,1)` | ⚠️ **확립된 근거 없음.** 위기정보 44종에도, 한전·통신 실서비스에도 우편 미수령은 *쓰이지 않음*(검증 결과). 사용자 제시 JAMA doi:10.1001/jama.2009.1109는 특정 실패. | **약** | **낮은 가중 강제 + 보조신호.** 단독 위험신호로 쓰지 말고 **다른 신호와 동시 발생 시에만 가산**. "외부에서 관찰 가능한 생활관리 실패 징후"(임대인·이웃·관리인이 고립을 발견하는 정황과 같은 계열)로만 약하게 서술. **가짜 인용 금지** — 못 찾으면 "확립된 출처 없는 보조 지표"로 정직하게 표기. |
| **사회적 고립** isolation | 복지관 6개월 0회 → 1/0 | Holt-Lunstad et al.(2010) PLoS Med 7(7):e1000316 (§7-4) + 고독사 실태조사(사회적 고립=고독사 직접 선행요인, §7-9) | **강(개념) / 이분지표는 약** | 사회적 단절↑→사망위험↑(OR 1.50)로 신호 채택 정당. **단 "혼자/0회" 같은 단순 이분지표는 메타분석상 효과 최약(OR 1.19)** → 과도한 가중 금지. "쓰던 사람이 갑자기 끊김"(궤적 변화)을 잡으면 더 타당 → §4 이상탐지와 연결. |

## 4. 작업 B — AI 에이전트 제거 + ML/DL 우선순위 모델 [피드백 ②]

### B-1. 문제 (가장 치명적 공격)
심사단: *"AI 에이전트를 왜 쓰나? API 호출 비싼데 공무원이 점수·그래프 보고 판단하면 되잖아."*
→ 맞는 지적. **에이전트라는 표현·구조를 전부 제거**한다(README에선 이미 'AI 에이전트' 표현을 뺐다고 되어 있으니 잔재 있으면 마저 제거).

### B-2. 방향 — ML을 "결정자"가 아니라 "대량 선별기(triage)"로 재정의 + 실서비스 선례

심사단 반박의 핵심 논리(이대로 코드/문서에 반영):

> 공무원이 점수·그래프를 보고 판단하는 건 **가구가 몇 개일 때만** 가능하다. 실제로는 1인 독거노인이 **수십만~수백만**, 신호가 5종, 시계열이 수 주치다. **현실 근거:** 노인복지사 1명이 평균 약 30명을 담당하는 상황에서 방문·전화만으로 위험을 조기 파악하기 어렵다(§7-10). 복지부 발굴 시스템조차 전체 약 500만 명을 사람이 다 못 보니 **빅데이터로 상위 약 4%만 자동 선별**해 통보한다(§7-7).
> **ML의 정당한 역할 = 규모(scale)에서의 자동 선별:** ① 사람이 다 못 보는 대량 가구에서 **급속·다변량 악화 케이스를 자동으로 골라내고**, ② 비슷한 궤적끼리 묶어 **방문 우선순위 그룹**을 만든다. ML은 위험을 *판정*하지 않는다(그건 G1 투명 점수 + G4 사람). ML은 **건초더미를 줄여** 적은 인력이 집중할 곳을 가리킨다.

**실서비스 선례(강력):** 한국전력·통신3사 안부살핌 서비스가 *바로 이 일*을 한다 — 전력·통신 패턴을 개인별로 학습·업데이트해 **변화 신호를 자동 포착**하고 담당자에게 알린다(§7-8). 즉 우리의 ML 방향은 **이미 공공에서 운영 중인 접근의 재현·교육용 PoC**다. ("억지로 AI 붙임"이 아니라 현실에 존재하는 모델.)

**추가 논거:** 단일 신호를 따로 보면 다 임계 미달이지만 **3개가 동시에 2주 만에 꺾이는** 가구가 있다. 그래프를 *한 개씩* 보는 사람은 이 *결합* 신호를 놓친다. 다변량 이상탐지는 결합 벡터에서 이걸 잡는다 → 기존 "임계값 vs 조합" 논리를 ML로 엄밀화한 것이자 "사람이 보면 되잖아"의 직접 반증.

### B-3. 권장 모델 (학술 적합 + PoC 현실성)

> 두 기능: **(1) 급속·다변량 악화 탐지**, **(2) 궤적 그룹화·우선순위.** 해석가능한 것부터 단계적으로.

**기능 1 — 급속·다변량 악화 탐지 (anomaly / change detection)**

*Tier 1 (먼저 구현 · 해석가능 · PoC 적합):*
- **신호별 변화탐지:** robust z-score(중앙값·MAD), EWMA 관리도, CUSUM, 단조감소는 Mann-Kendall 검정 / Theil-Sen 기울기 → "어느 신호가 언제 급변했는가"를 설명가능하게.
- **다변량 이상점수:** **Isolation Forest**(Liu, Ting & Zhou 2008)를 *최근 주간 델타/기울기 피처 벡터*에 적용. 보조로 Local Outlier Factor(Breunig et al. 2000), robust Mahalanobis(MCD).
- **변화점 탐지(선택):** PELT / Binary Segmentation(Killick et al. 2012)로 "급락 시점" 추정.

*Tier 2 (DL · 여유 시 · 확장 경로):*
- **LSTM-Autoencoder**(Malhotra et al. 2016) — 다변량 시계열 재구성오차를 이상점수로. 다변량 TS 이상탐지 표준 인용. (더 최신: USAD(KDD 2020), Anomaly Transformer(ICLR 2022).)
- ⚠️ **현실 경고(문서에 명시):** 현재 데이터는 합성 74가구 × 8주. 이 규모에서 DL은 과적합·시연용에 가깝다. **Tier 1을 본체로, DL은 "실데이터 확장 경로"로 포지셔닝**(README '한계와 책임' 기조 유지).

**기능 2 — 궤적 그룹화 + 우선순위 (clustering + ranking)**
- **클러스터링:** 정규화 궤적 피처(신호별 기울기·최근성 가중 델타)에 **k-means** 또는 **DBSCAN**(Ester et al. 1996; 밀도기반→이상 그룹 분리). 형태 비교 필요시 **DTW + k-means**(tslearn). → 사용자가 말한 "우선순위 그룹화".
- **우선순위 산출:** **블랙박스 랭커 금지**(합성 라벨 지도학습 랭킹은 순환적·G1 위반). 대신 **투명 점수 융합**: `priority = f(riskFI, anomalyScore, 추세상승폭, 반복통보)` — 가중·규칙을 명시·조정가능하게. 기존 `priority.ts`의 "추세 반영 우선순위"를 이 융합으로 확장.

### B-4. 아키텍처 — 오프라인 학습 → JSON 산출 (client-only 유지)
1. **권장(기본):** `scripts/`에 **Python ML 파이프라인**(scikit-learn 등) 추가. 합성 데이터에 anomaly score·cluster·change-point를 *오프라인 계산*해 `households.json`에 필드 추가(`anomalyScore`, `clusterId`, `changePoints` 등). UI는 그 값을 *표시만*.
   - 장점: 진짜 ML 라이브러리, 재현성(시드 고정), client-only 유지, G1/G4 자연 충족(런타임에 모델이 점수 안 만듦).
   - `scripts/run_ml.py`(+`requirements.txt`) 추가, `gen:data` 흐름에 통합 또는 별도 스텝.
2. **선택(인터랙티브 데모):** TensorFlow.js / ONNX Runtime Web 인브라우저 추론. (비용↑, 우선순위 낮음.)

### B-5. UI/UX 반영
- **방문계획 탭(③):** 우선순위 정렬에 `anomalyScore`·"급속악화" 배지. 클러스터(궤적 그룹)별 카드. "왜 위로 왔나 = FI점수 + 급속악화 + 반복통보" 설명 칩.
- **분석 탭(①):** "임계값 vs 조합" 대조에 **"사람 눈으로는 못 잡는 결합 급락"** 사례를 이상탐지로 시각화(각 신호 임계 미달인데 결합 이상도↑).
- **에이전트 잔재 제거:** 'AI 에이전트' 표현·아이콘·카피가 코드/문서/UI에 남아있으면 'ML 보조 선별' 또는 '근거 서술'로 교체.

8. **[전력·통신 안부살핌 실서비스]** 한국전력 + 이동통신 3사 'AI 안부살핌' 서비스 / 서울시 스마트플러그 사업 — 전력·통신 패턴을 개인별 학습·업데이트해 변화신호 자동 포착→담당자 알림. 한전이 "전력 급감만 → 생활패턴 전반 분석"으로 발전. *용도:* power·telecom 신호 정당 + §4 패턴기반 이상탐지의 실서비스 선례. (보도자료·지자체 발표 확인 후 인용.)
9. **[고독사 실태]** 2022 고독사 실태조사 등 — 사회적 고립이 고독사 직접 선행요인, 50~60대 남성 비중↑, 가족 발견 감소·제3자(임대인·이웃) 발견 증가. *용도:* isolation 신호 맥락 + 문제 동기. *강도: 맥락 근거.*
10. **[돌봄 인력 한계]** 노인복지사 1인당 평균 약 30명 담당 → 방문·전화만으로 조기 파악 어려움(언론·보고서 인용). *용도:* **"공무원이 그냥 보면 되잖아"에 대한 핵심 반박(피드백 ②).** (1차 통계/보고서 확인 후 인용.)

**[ML 모델 출처]** Liu, Ting & Zhou (Isolation Forest, 2008); Breunig et al. (LOF, 2000); Killick et al. (PELT, 2012); Ester et al. (DBSCAN, 1996); Malhotra et al. (LSTM-AE, 2016). — 정확 서지는 Claude Code가 확인해 추가.

## 8. 열린 질문 / 진행 메모 (Claude Code는 막히면 여기에 기록)
- **우편 신호:** 확립된 출처 없음 확정 → 보조·낮은 가중으로 처리(강한 인용 금지). 대안 신호 발견 시에만 상향.
- **통신 신호 개선안:** 현재 변수는 "연체(arrears)". 실서비스는 "통신 *사용패턴* 변화"를 쓴다 → 데이터 생성/모델에 통신 사용패턴 변화 신호를 추가하면 근거가 더 강해짐(선택).
- **DL(Tier 2):** 합성 74가구×8주 규모상 실학습보다 "확장 경로" 문서화 권장. 실제 학습/시연 여부는 판단.
- **인증:** PoC는 mock 권장(C-4). 실제 백엔드(Supabase 등)까지 갈지는 범위 판단.
- **임계값 분위수:** 상위 몇 %로 할지 도메인 판단 필요(복지부 "상위 약 4%" 참고 가능).
- **1차 출처 확인:** §7-7~10은 언론/보도자료 경유. Claude Code가 복지부·한전·통계 **1차 출처**를 찾아 정식 인용으로 교체(인용 신뢰도↑).

### 한 줄 요약 (Claude Code용)
> **점수의 *형식*은 Frailty Index(결핍누적)로, 신호의 *선택*은 한국 복지 발굴 위기정보 체계·실서비스(한전·통신3사 안부살핌)로 정당화한다. ML은 "사람이 전수로 못 보는 대량 가구에서 급속·다변량 악화를 골라내는 선별기"로 (에이전트가 아니라) 재정의하며 — 이는 이미 운영 중인 공공서비스의 진화 경로와 동일하다. 데스크톱은 유지한 채 모바일·PWA·데모 로그인을 추가한다. 모든 수치는 출처/가정/학습 중 하나로 분류하고, 약한 근거(특히 우편)에는 가짜 인용을 달지 않는다.**
